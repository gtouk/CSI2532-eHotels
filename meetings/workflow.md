# Workflow Git

## Branches
- `main` : branche stable du projet
- `Naomi` : branche de travail de Naomi
- `Gilly` : branche de travail de Gilly
- `Martial` : branche de travail du 3e membre
- `Antoine` : branche de travail du 4e membre

## Règles
- Chaque membre travaille uniquement sur sa propre branche.
- La branche `main` doit toujours rester stable.
- Avant d’ajouter du code dans `main`, il faut vérifier que le code fonctionne.
- Les fichiers doivent rester bien organisés par dossier.
- Les décisions techniques doivent être notées dans `meetings/`.

## Commits
Utiliser des messages clairs, par exemple :
- `add client table`
- `create reservation page`
- `update ER diagram`
- `fix SQL constraints`
